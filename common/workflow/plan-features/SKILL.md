# Plan Features (스택 중립)

## 먼저 읽기
- `CLAUDE.md`, `.tasks/backlog.md` — 기존 컨텍스트
- **현 프로젝트의 conventions 스킬**(`ts-vite-react` / `ts-next` / `flutter-app` / `py-fastapi` 등) — **태스크 태그 체계**를 여기서 가져온다

## 산출
1. **화면/메뉴 계층 구조** (트리)
2. **레이어별 태스크 분해** — 태그는 **현 conventions 스킬의 `## 태스크 태그` 섹션을 그대로 따른다**(스택마다 다르므로 여기 나열하지 않는다 — SSOT는 conventions). 각 태스크는 2~4시간 단위.
3. **우선순위** P0(핵심)/P1(중요)/P2(부가)
4. **wave·의존성·QA 부기** — 각 태스크에 `wave`(동시에 돌려도 되는 묶음 번호)·`deps`(선행 태스크)·`QA`(완료를 확인할 사용자 관찰 시나리오 1줄)를 붙인다. PM이 "순차 vs 병렬"과 "무엇으로 완료를 판정할지"를 이 정보로 결정한다.
5. `.tasks/backlog.md`에 기록:

```md
## [기능명] — YYYY-MM-DD 등록
### 📱 화면/메뉴 구조
### 📋 태스크 목록
#### P0 — 핵심
- [ ] [태그] 설명 | 1h | wave:1 | deps:없음 | QA: <사용자가 무엇을 보면 완료인가 1줄>
```
> 같은 `wave`·`deps:없음`끼리는 병렬 가능(파일 비중첩 시), 선행이 있으면 순차.
