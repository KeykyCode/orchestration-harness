# Checkpoint — 상태 외재화 & 세션 리셋 계약 (스택 중립)

> 세션을 오래 살리는 대신 **짧게 살리고 상태를 파일로 넘긴다.** 컨텍스트는 관리하는 자원이다.
> 상태 영속화(파일)와 컨텍스트 위생(리셋)은 **별개 축** — 파일에 잘 적어도 오염된 라이브 세션은 안 나아진다.

## 1. 언제
- 작업 청크(기능 1개·검증 1라운드) 완료 시.
- 세션이 길어져 판단이 흐려지는 징후: 같은 파일 반복 오독·앞서 정한 결정 망각·검증 출력 누적.

## 2. resume.md 갱신 (핵심)
`.tasks/resume.md`의 각 섹션을 **현재 상태로 덮어쓴다**(append 아님, 최신 스냅샷만). 히스토리는 넣지 않는다.
- **"다음 1수"**: `iterate`를 돌렸으면 **그 결과를 옮겨 담는다**(여기서 재도출하지 않음). 역할 구분 — `iterate`=진척 분석(→ iterate-log 히스토리), `checkpoint`=리셋용 스냅샷(→ resume 덮어쓰기).
- **"불변식/함정"**: **이 세션에서 발견한 것만**. 영구 규칙(예: "인증은 서버 판단")은 resume가 아니라 conventions/CLAUDE.md/메모리로 **래칫**한다.

## 3. 아카이브 분리
완료 항목은 `done.md`, 진척 로그는 `iterate-log.md`로 이동. resume.md엔 남기지 않는다(짧게 유지).

## 4. 리셋 권고
사용자에게 안내: *"resume.md에 상태를 저장했습니다. `/clear` 후 '이 프로젝트 resume.md 읽고 이어가줘'로 새 세션에서 이어가면 컨텍스트가 깨끗합니다."*
> Claude Code의 자동 compaction보다 이 **큐레이션 리셋**이 더 신뢰도 높다(요약이 뭉개는 것 방지). 완전 자동화는 hook 필요 — 현재는 권고까지.

## 5. 재시작 규약
새 세션의 첫 행동은 `resume.md` **하나만** 읽고 그로부터 재구성하는 것. 옛 세션의 인라인 기억에 의존 금지.
